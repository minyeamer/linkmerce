# AI Coding Guidelines (Mandatory)

이 문서는 AI 어시스턴트가 본 프로젝트에서 작업할 때 반드시 준수해야 하는 **강제 지침**입니다. 모든 작업은 아래 규칙을 최우선적으로 따릅니다.

1. **파일 수정 원칙**: 모든 파일 수정(로그 파일 포함)은 반드시 `replace_string_in_file` 또는 `insert_edit_into_file` 도구만 사용한다. `run_in_terminal`을 통해 파이썬 스크립트나 명령어로 파일을 직접 조작하는 행위는 **엄격히 금지**한다.
2. **대용량 파일 처리**: 파일 크기가 커서 전체 읽기가 어려운 경우, 임의로 생략하지 말고 어느 범위를 읽어야 할지 사용자에게 명확히 확인을 구한다.
3. **작업 로그 기록 (필수)**: 작업 완료 시 `docs/chat.log`에 채팅 내역과 의사결정 과정을 반드시 기록한다. 이 파일은 시간 순으로 하단에 누적되는 형식이므로, 기록 시 최하단 50줄을 참고하여 일관성을 유지한다. 기록 내용을 확인할 때는 인코딩 문제를 방지하기 위해 반드시 `python -c "with open('.../docs/chat.log', encoding='utf-8') as f: ..."` 형식을 사용한다. 사용자의 명시적 거부가 없는 한 세션 종료 시 **항상 수행**한다.
4. **커밋 메시지 출력**: 작업 완료 후 변경 사항을 요약한 Git 커밋 메시지를 작성하여 출력한다. 단, 사용자의 별도 지시가 없는 한 실제 커밋 명령(`git commit`)을 직접 수행하지는 않는다.
